HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The body element is an example of a semantic HTML element.
  • In Semantic HTML elements the name of the element describes its content and role.
  • Examples of common HTML elements include h1, p, ul, ol, li, img.

CSS

  • CSS is used in styling a webpage
  • There are three ways to style a webpage using CSS:
    inline CSS, internal CSS stylesheet and external CSS stylesheet
  • External stylesheets have to be linked to the html file by a link element within the head element
  • A CSS rule consists of a selector (which can be an html element or defined class or id of an element),
    a property and its assigned value enclosed in curly braces such as color: blue.
  • An important concept to understand in applying CSS rules is to think of each html element as a box.
    This is referred to as the CSS box model.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a version control software that lets us manage and keep track of ech version of our software.
  • Git is great for tarcking changes locally in our computer, but to realise its full potential
    it should be used in conjuction with Github or other git-based repository hosting services.
  • Git commands are always prefaced with git and has to be employed at the CLI.
    Egs include git -init, git clone, git status, git add, git commit -m, git log, etc
  • The Github workflow starts from creating a feature branch
  • This is followed by making changes to your branch via commits and push to the remote repository.
  • Once you have completed work on your feature branch, create a pull request.
    This will allow your teammates review your code and to verify that it is safe to merge with the default branch.
  • Once pull request is approved and your branch has been merged, it is good practice to delete your branch as
    it indicates that the work on that branch is complete and prevents you or others from accidentally using old branches.
  • Finally, navigate back to the main or default branch and update the local repository in your machine with the new merged-in feature.

JavaScript

  • JavaScript adds interactivity to a webpage by making static pages dynamic
  • To add JS logic to our webpage we must link the file containing it to our html file.
  • Note that whereas the css file is linked in the head element of the HTML file,
    the JS file is linked in the very bottom of the body element just before the closing body tag.
  • A script element is used to link the js file with the path of the file set to the src attribute.
  • A variable in JavaScript is simply a container that allows us to store data or a value within it.
  • Control flow is the order in which the computer executes code in a script,
    in JavaScript this order is top to bottom and left to right.
  • Conditional statements and loops can be used to control the
    order in which code is executed or whether it gets executed at all.
  • If-Else statements are one example of conditional statements while for loops are an example of loops.
  • An array is a single variable(container) that can be used to hold a grouo of data.
  • In JS like in most programming languages, arrays are always zero-indexed,
    ie the first element in the array is located at index 0.
  • A for-loop provide an easier way to iterate through all the elements of our array by writing less code.
  • Functions are building blocks of JavaScript. When we create a function we are
    defining a block of code that performs a specific task and giving it a unique name.
  • However, unlike conditionals and loops, a function won't execute automatically without us calling it.
